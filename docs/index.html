QuantaTissu Project

A multi-faceted project to build an autonomous AI software engineering agent from scratch.

Introduction

QuantaTissu is a project centered around building an autonomous AI software engineering agent. It is comprised of several key components built from the ground up to be both an educational tool and the foundation for a sophisticated coding assistant.

The project includes:

  • QuantaTissu: A transformer-based language model serving as the agent's core reasoning engine.
  • TissLang: A declarative language for orchestrating the agent's actions in a structured manner.
  • TissDB: A high-performance NoSQL database for persistence.
  • Tissu Sinew: A native C++ connector for TissDB.

The QuantaTissu Language Model

QuantaTissu is a decoder-only transformer model built in Python with NumPy. It features a full training and inference pipeline with modern, efficient components.

Implemented Features:

  • Architecture: A configurable N-layer transformer with multi-head self-attention and causal masking.
  • Tokenization: A trainable Byte-Pair Encoding (BPE) subword tokenizer.
  • Inference: Supports batched inference, KV Caching for speed, and advanced sampling methods (greedy, top-k, nucleus).
  • Training: A full pipeline with an Adam optimizer, cross-entropy loss, learning rate scheduling, and model checkpointing.

Future work will focus on architectural upgrades like SwiGLU and RoPE, and advanced techniques like quantization and LoRA for fine-tuning.

TissLang: The Agentic Language

TissLang is a structured, human-readable language designed to direct the agent's actions safely and predictably. It includes a parser and an execution engine to interpret scripts.

Implemented Features:

  • Core Syntax: `TASK` and `STEP` blocks to define and organize workflows.
  • Commands: `WRITE` (to files), `RUN` (shell commands), and `READ` (file content).
  • Assertions: `ASSERT` to verify command outcomes, such as exit codes and stdout content.

Future plans include adding control flow constructs like `IF/ELSE` and enabling the agent to generate its own TissLang scripts as plans for human review.

TissDB: A High-Performance NoSQL Database

TissDB is a lightweight NoSQL database built in C++ with an LSM-Tree storage engine, a JSON-like document model, and a SQL-like query language (TissQL). It ensures durability with a Write-Ahead Log (WAL) and provides a RESTful API for interaction.

Limitations and Future Work:

TissDB is under active development. Current limitations include being primarily in-memory (relying on WAL replay on startup), non-persistent B-Tree indexes, and incomplete ACID transaction support. Future work will focus on improving persistence and transactionality.

Tissu Sinew: C++ Connector for TissDB

Tissu Sinew is the official C++ connector for TissDB. It provides a high-performance, modern C++ interface designed for safety and ease of use. Key features include a thread-safe connection pool, RAII-based session and transaction management to prevent resource leaks, and a clear exception hierarchy for robust error handling.

Ecological Awareness

A core principle of the project is to promote sustainable AI development through a three-pronged strategy:

  1. Model Efficiency: Optimizing the model for lower energy consumption.
  2. Green Code Generation: Training the agent to recognize and suggest energy-efficient code.
  3. Impact Transparency: Building tools in TissLang to estimate the carbon footprint of code changes.
